Fixes in Xl Release 7.0.2
      Bug fixes
    • REL-4388 - Variable order in user input task issue more

      The ordering of variables in a user input task is not preserved. It will always follow the same order in the release's Variable configuration.

      To reproduce:
      1. Create a release
      2. Add a few variables to the release
      3. Add a User Input task
      4. Add the variables to the task in random order and save the task
      5. Exit the task and enter the task again, the ordering would be altered to be the same as how it is in the release's Variable configuration

      Regression in XLR 6.x.x. Tested negative in 5.0.1

    • REL-4558 - Disable 'Complete' action for User Input Task when required variables are left empty
